1. BREADCRUMBS (Migas de Pan)
-----------------------------------------------------------
¿QUÉ SON?
  Las breadcrumbs son enlaces de navegación que muestran la ruta
  desde la página principal hasta donde estás ahora.

¿DÓNDE SE AGREGARON?
  ✓ catalogo.html: Inicio > Catálogo
  ✓ retros.html: Inicio > Catálogo > Camisetas Retro
  ✓ contacto.html: Inicio > Contacto

EJEMPLO DE CÓDIGO:
  <nav aria-label="breadcrumb">
      <div class="container">
          <ol class="breadcrumb">
              <li class="breadcrumb-item">
                  <a href="index.html"><i class="fas fa-home"></i> Inicio</a>
              </li>
              <li class="breadcrumb-item active" aria-current="page">
                  Catálogo
              </li>
          </ol>
      </div>
  </nav>

BENEFICIOS:
  • Mejora la navegación del usuario
  • Muestra claramente dónde está el usuario
  • Facilita volver a páginas anteriores
  • Mejora el SEO

2. BOTÓN "VOLVER ARRIBA" (Back to Top)
-----------------------------------------------------------
¿QUÉ ES?
  Un botón flotante que aparece cuando haces scroll hacia abajo
  y te lleva instantáneamente al inicio de la página.

CARACTERÍSTICAS:
  • Solo aparece después de scrollear 300px
  • Posición fija en la esquina inferior derecha
  • Animación suave al aparecer/desaparecer
  • Efecto hover que lo eleva
  • Scroll suave al hacer click

CÓDIGO CSS:
  #backToTop {
      position: fixed;
      bottom: 30px;
      right: 30px;
      background-color: #0d6efd;
      color: white;
      border-radius: 50%;
      width: 50px;
      height: 50px;
      opacity: 0;
      visibility: hidden;
  }
  
  #backToTop.show {
      opacity: 1;
      visibility: visible;
  }

CÓDIGO JAVASCRIPT:
  window.addEventListener('scroll', function() {
      if (window.pageYOffset > 300) {
          backToTopButton.classList.add('show');
      } else {
          backToTopButton.classList.remove('show');
      }
  });

BENEFICIOS:
  • Mejora la experiencia en páginas largas
  • Ahorra tiempo al usuario
  • Aspecto profesional y moderno

3. SCROLL SUAVE
-----------------------------------------------------------
¿QUÉ ES?
  Hace que cuando navegas entre secciones o vuelves arriba,
  el movimiento sea animado y suave en lugar de instantáneo.

IMPLEMENTACIÓN:
  CSS (una sola línea):
    html {
        scroll-behavior: smooth;
    }

  JavaScript (para el botón):
    window.scrollTo({
        top: 0,
        behavior: 'smooth'
    });

BENEFICIOS:
  • Experiencia más agradable
  • Reduce el mareo del usuario
  • Aspecto profesional

5. ANIMACIONES DE ENTRADA (Fade-in)
-----------------------------------------------------------
¿QUÉ SON?
  Las tarjetas de productos aparecen con una animación suave
  cuando cargas la página, en lugar de aparecer todas de golpe.

CARACTERÍSTICAS:
  • Animación de abajo hacia arriba (translateY)
  • Fade-in (aparición gradual de opacidad)
  • Delay progresivo (cada card aparece un poco después)
  • Duración: 0.8 segundos

CÓDIGO CSS:
  @keyframes fadeInUp {
      from {
          opacity: 0;
          transform: translateY(30px);
      }
      to {
          opacity: 1;
          transform: translateY(0);
      }
  }
  
  .card {
      animation: fadeInUp 0.8s ease-out;
  }
  
  /* Delay progresivo */
  .col-lg-3:nth-child(1) .card { animation-delay: 0.1s; }
  .col-lg-3:nth-child(2) .card { animation-delay: 0.2s; }
  .col-lg-3:nth-child(3) .card { animation-delay: 0.3s; }
  ...

BENEFICIOS:
  • Captura la atención del usuario
  • Aspecto más dinámico y moderno
  • Mejora la percepción de calidad

6. INTERACTIVIDAD CON JAVASCRIPT
-----------------------------------------------------------
Se creó el archivo js/main.js con múltiples funcionalidades:

A) CONTADOR DE CARRITO FUNCIONAL
   • Al hacer click en "Añadir al Carrito"
   • El botón cambia a "¡Agregado!" con check verde
   • El contador del carrito incrementa
   • El icono del carrito hace shake (tiembla)
   • Después de 2 segundos vuelve al estado original

B) MEJORA DEL CARRUSEL
   • Pausa automática al pasar el mouse
   • Reanuda al quitar el mouse
   • Mejores transiciones visuales

C) VALIDACIÓN DE FORMULARIO
   • Valida campos obligatorios en contacto
   • Muestra alertas si falta información
   • Simula envío exitoso

D) BÚSQUEDA PREPARADA
   • Intercepta el formulario de búsqueda
   • Muestra alerta informativa
   • Lista para conectar con backend

EJEMPLO DE CÓDIGO (Contador de Carrito):
  button.addEventListener('click', function(e) {
      e.preventDefault();
      
      // Incrementar contador
      cartCount++;
      const cartBadge = document.querySelector('.navbar-nav .badge');
      if (cartBadge) {
          cartBadge.textContent = cartCount;
      }
      
      // Feedback visual
      button.innerHTML = '<i class="fas fa-check"></i> ¡Agregado!';
      button.classList.add('btn-success');
      
      // Volver al estado original
      setTimeout(() => {
          button.innerHTML = '<i class="fas fa-cart-plus"></i> Añadir';
          button.classList.remove('btn-success');
      }, 2000);
  });

BENEFICIOS:
  • Feedback inmediato al usuario
  • Sensación de sitio "vivo" y dinámico
  • Mejora la experiencia de usuario
  • Preparado para backend futuro

=========================================================
 💡 TIPS PROFESIONALES
=========================================================

1. BREADCRUMBS:
   • Siempre incluye el icono de home al inicio
   • El último elemento no debe ser clickeable (active)
   • Usa atributo aria-label="breadcrumb" para accesibilidad

2. BOTÓN VOLVER ARRIBA:
   • No lo muestres inmediatamente (espera 300px de scroll)
   • Usa position: fixed para que siempre sea visible
   • Añade z-index alto para que esté sobre todo

3. ANIMACIONES:
   • No abuses de las animaciones (pueden marear)
   • Usa duración entre 0.3s - 0.8s (más lento es molesto)
   • Aplica ease-out para animaciones naturales

4. JAVASCRIPT:
   • Usa addEventListener en lugar de onclick en HTML
   • Previene el comportamiento por defecto con e.preventDefault()
   • Valida que los elementos existen antes de manipularlos

5. PERFORMANCE:
   • Las animaciones CSS son más rápidas que JavaScript
   • Usa transform en lugar de top/left para movimiento
   • opacity es más eficiente que cambiar visibility
