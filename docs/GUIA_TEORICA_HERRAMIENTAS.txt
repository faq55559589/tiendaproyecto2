# 1. 🗄️ MYSQL - BASE DE DATOS RELACIONAL
# ================================================================

## ¿QUÉ ES MYSQL?
MySQL es un sistema de gestión de bases de datos relacionales (RDBMS) de código abierto.
Es como un "almacén digital" donde guardamos toda la información de tu tienda.

## 🎯 ¿PARA QUÉ SIRVE?
- **Almacenar datos**: Usuarios, productos, pedidos, etc.
- **Organizar información**: En tablas relacionadas entre sí
- **Consultar datos**: Buscar productos, verificar usuarios
- **Mantener integridad**: Evitar datos duplicados o incorrectos

## 🏗️ ESTRUCTURA BÁSICA:
```
📊 BASE DE DATOS: fankit_db
├── 👥 Tabla: usuarios
│   ├── id (clave primaria)
│   ├── nombre
│   ├── email
│   └── contraseña_encriptada
├── 🛍️ Tabla: productos
│   ├── id
│   ├── nombre
│   ├── precio
│   └── stock
└── 🛒 Tabla: pedidos
    ├── id
    ├── usuario_id (clave foránea)
    └── producto_id (clave foránea)
```

## 🔧 COMANDOS BÁSICOS:
```sql
-- Crear base de datos
CREATE DATABASE fankit_db;

-- Usar base de datos
USE fankit_db;

-- Crear tabla
CREATE TABLE usuarios (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    contraseña VARCHAR(255) NOT NULL
);

-- Insertar datos
INSERT INTO usuarios (nombre, email, contraseña) 
VALUES ('Juan', 'juan@email.com', 'password123');

-- Consultar datos
SELECT * FROM usuarios WHERE email = 'juan@email.com';
```

## ⚡ VENTAJAS:
- ✅ **Rápido**: Optimizado para consultas complejas
- ✅ **Confiable**: Usado por millones de aplicaciones
- ✅ **Escalable**: Maneja grandes volúmenes de datos
- ✅ **Estándar**: SQL es universal

## ⚠️ DESVENTAJAS:
- ❌ **Complejo**: Requiere conocimiento de SQL
- ❌ **Estructurado**: Datos deben seguir esquema fijo

# 6. 🔐 BCRYPT - ENCRIPTACIÓN DE CONTRASEÑAS
# ================================================================

## ¿QUÉ ES BCRYPT?
bcrypt es una librería para encriptar contraseñas de forma segura.
Es como un "candado digital" para proteger datos sensibles.

## 🎯 ¿PARA QUÉ SIRVE?
- **Encriptar contraseñas**: Nunca guardar texto plano
- **Verificar login**: Comparar contraseña ingresada
- **Seguridad**: Resistente a ataques de fuerza bruta
- **Salt automático**: Cada contraseña tiene sal única

## 🔒 ¿POR QUÉ ES IMPORTANTE?
```
❌ MALO - Contraseña en texto plano:
usuario: juan@email.com
contraseña: "123456" ← ¡PELIGROSO!

✅ BUENO - Contraseña encriptada:
usuario: juan@email.com
contraseña: "$2b$10$N9qo8uLOickgx2ZMRZoMye..." ← SEGURO
```

## 🔧 CÓDIGO BÁSICO:
```javascript
const bcrypt = require('bcrypt');

// Encriptar contraseña (registro)
const saltRounds = 10;
const password = 'miContraseña123';
const hashedPassword = await bcrypt.hash(password, saltRounds);

// Verificar contraseña (login)
const isValid = await bcrypt.compare(password, hashedPassword);
```

## 🛡️ FLUJO DE SEGURIDAD:
```
1. Usuario ingresa contraseña
2. bcrypt genera hash único
3. Se guarda hash en base de datos
4. En login, se compara hash
5. Si coincide → acceso permitido
```

## ⚡ CARACTERÍSTICAS:
- **Salt automático**: Cada hash es único
- **Coste configurable**: Más rondas = más seguro
- **Lento intencionalmente**: Resistente a ataques
- **Estándar de industria**: Usado por millones de apps

# 7. 🍪 JWT - AUTENTICACIÓN CON TOKENS
# ================================================================

## ¿QUÉ ES JWT?
JWT (JSON Web Token) es un estándar para crear tokens de autenticación.
Es como un "pase de acceso" digital que expira.

## 🎯 ¿PARA QUÉ SIRVE?
- **Autenticación**: Verificar que el usuario está logueado
- **Autorización**: Controlar qué puede hacer cada usuario
- **Sesiones**: Mantener login sin cookies
- **APIs**: Autenticación en aplicaciones móviles

## 🏗️ ESTRUCTURA DEL TOKEN:
```
Header.Payload.Signature
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOjEyMywibmFtZSI6Ikp1YW4ifQ.signature
```

## 🔧 CÓDIGO BÁSICO:
```javascript
const jwt = require('jsonwebtoken');

// Crear token (login)
const token = jwt.sign(
    { userId: 123, email: 'juan@email.com' },
    'mi_clave_secreta',
    { expiresIn: '24h' }
);

// Verificar token (middleware)
const decoded = jwt.verify(token, 'mi_clave_secreta');
```

## 🔄 FLUJO DE AUTENTICACIÓN:
```
1. Usuario hace login
2. Servidor verifica credenciales
3. Servidor crea JWT
4. Cliente guarda JWT
5. Cliente envía JWT en cada request
6. Servidor verifica JWT
7. Si válido → acceso permitido
```

## 🛡️ MIDDLEWARE DE AUTENTICACIÓN:
```javascript
function authenticateToken(req, res, next) {
    const token = req.headers['authorization'];
    
    if (!token) {
        return res.status(401).json({ error: 'Token requerido' });
    }
    
    try {
        const decoded = jwt.verify(token, 'mi_clave_secreta');
        req.user = decoded;
        next();
    } catch (error) {
        return res.status(403).json({ error: 'Token inválido' });
    }
}
```

# 8. 🔄 CORS - SEGURIDAD ENTRE DOMINIOS
# ================================================================

## ¿QUÉ ES CORS?
CORS (Cross-Origin Resource Sharing) permite que tu frontend 
se comunique con tu backend desde diferentes dominios.

## 🎯 ¿PARA QUÉ SIRVE?
- **Comunicación frontend-backend**: Desde localhost:3000 a localhost:5000
- **APIs públicas**: Permitir acceso desde cualquier dominio
- **Seguridad**: Controlar qué dominios pueden acceder
- **Desarrollo**: Evitar errores de CORS en desarrollo

## 🌐 PROBLEMA SIN CORS:
```
❌ ERROR: Access to fetch at 'http://localhost:5000/api/productos' 
from origin 'http://localhost:3000' has been blocked by CORS policy
```

## 🔧 SOLUCIÓN CON CORS:
```javascript
const cors = require('cors');

// Permitir todos los orígenes (desarrollo)
app.use(cors());

// Configuración específica (producción)
app.use(cors({
    origin: ['http://localhost:3000', 'https://fankit.com'],
    credentials: true
}));
```

## 🛡️ CONFIGURACIÓN SEGURA:
```javascript
const corsOptions = {
    origin: function (origin, callback) {
        // Permitir requests sin origin (mobile apps, etc.)
        if (!origin) return callback(null, true);
        
        const allowedOrigins = [
            'http://localhost:3000',
            'https://fankit.com'
        ];
        
        if (allowedOrigins.indexOf(origin) !== -1) {
            callback(null, true);
        } else {
            callback(new Error('No permitido por CORS'));
        }
    }
};
```

# 9. 📁 MULTER - SUBIDA DE ARCHIVOS
# ================================================================

## ¿QUÉ ES MULTER?
Multer es un middleware para manejar la subida de archivos 
en aplicaciones Node.js.

## 🎯 ¿PARA QUÉ SIRVE?
- **Subir imágenes**: Fotos de productos
- **Validar archivos**: Tamaño, tipo, formato
- **Almacenar archivos**: En servidor o cloud
- **Procesar imágenes**: Redimensionar, optimizar

## 🔧 CÓDIGO BÁSICO:
```javascript
const multer = require('multer');

// Configuración de almacenamiento
const storage = multer.diskStorage({
    destination: function (req, file, cb) {
        cb(null, 'uploads/');
    },
    filename: function (req, file, cb) {
        cb(null, Date.now() + '-' + file.originalname);
    }
});

const upload = multer({ storage: storage });

// Ruta para subir archivos
app.post('/api/upload', upload.single('imagen'), (req, res) => {
    res.json({ archivo: req.file.filename });
});
```

## 🛡️ VALIDACIÓN DE ARCHIVOS:
```javascript
const upload = multer({
    storage: storage,
    limits: {
        fileSize: 5 * 1024 * 1024 // 5MB máximo
    },
    fileFilter: function (req, file, cb) {
        if (file.mimetype.startsWith('image/')) {
            cb(null, true);
        } else {
            cb(new Error('Solo se permiten imágenes'));
        }
    }
});
```

## 📊 FLUJO DE SUBIDA:
```
1. Usuario selecciona imagen
2. Frontend envía archivo
3. Multer valida archivo
4. Archivo se guarda en servidor
5. Se devuelve URL de la imagen
6. URL se guarda en base de datos
```

# 13. 🐘 POSTGRESQL - BASE DE DATOS AVANZADA (ALTERNATIVA)
# ================================================================

## ¿QUÉ ES POSTGRESQL?
PostgreSQL es una base de datos relacional avanzada 
con características de No-SQL.

## 🎯 ¿CUÁNDO USAR POSTGRESQL?
- **Aplicaciones complejas**: Muchas relaciones
- **Datos estructurados**: Esquema bien definido
- **Transacciones**: ACID compliance
- **Escalabilidad**: Manejo de grandes volúmenes

## 🏗️ CARACTERÍSTICAS AVANZADAS:
```sql
-- Tipos de datos avanzados
CREATE TABLE productos (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(100),
    precio DECIMAL(10,2),
    especificaciones JSONB,
    fecha_creacion TIMESTAMP DEFAULT NOW()
);

-- Consultas complejas
SELECT p.*, c.nombre as categoria
FROM productos p
JOIN categorias c ON p.categoria_id = c.id
WHERE p.precio > 100
ORDER BY p.fecha_creacion DESC;
```

## 🔧 CÓDIGO BÁSICO:
```javascript
const { Pool } = require('pg');

const pool = new Pool({
    user: 'postgres',
    host: 'localhost',
    database: 'fankit',
    password: 'password',
    port: 5432,
});

// Consulta
const result = await pool.query(
    'SELECT * FROM usuarios WHERE email = $1',
    [email]
);
```

## ⚡ VENTAJAS:
- ✅ **Potente**: Características avanzadas

## 🏆 STACK RECOMENDADO PARA FANKIT:

### 🥇 **OPCIÓN 1 - STACK TRADICIONAL:**
- **Base de datos**: MySQL
- **Backend**: Node.js + Express
- **Autenticación**: JWT + bcrypt
- **Archivos**: Multer
- **Testing**: Postman

### 🥈 **OPCIÓN 2 - STACK MODERNO:**
- **Base de datos**: PostgreSQL
- **Backend**: Node.js + Express
- **ORM**: Prisma o Sequelize
- **Autenticación**: JWT + bcrypt
- **Testing**: Jest + Supertest

### 🥉 **OPCIÓN 3 - STACK RÁPIDO:**
- **Base de datos**: MongoDB
- **Backend**: Node.js + Express
- **ORM**: Mongoose
- **Autenticación**: JWT + bcrypt
- **Testing**: Postman

## 🚀 PRÓXIMOS PASOS:

1. **Configurar MySQL** ✅
2. **Instalar Node.js** ⏳
3. **Crear estructura backend** ⏳
4. **Implementar autenticación** ⏳
5. **Conectar con frontend** ⏳
6. **Testing y deployment** ⏳
