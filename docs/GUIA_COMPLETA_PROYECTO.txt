🗺️ BREADCRUMBS (Migas de Pan)
  
  Ejemplos:
  • catalogo.html: Inicio > Catálogo
  • retros.html: Inicio > Catálogo > Camisetas Retro
  • contacto.html: Inicio > Contacto
  
  Código HTML:
  
  <nav aria-label="breadcrumb">
      <ol class="breadcrumb">
          <li class="breadcrumb-item">
              <a href="index.html">
                  <i class="fas fa-home"></i> Inicio
              </a>
          </li>
          <li class="breadcrumb-item active">Catálogo</li>
      </ol>
  </nav>

🔼 BOTÓN "VOLVER ARRIBA"
  
  Características:
  • Aparece solo después de scrollear 300px
  • Posición fija en esquina inferior derecha
  • Animación suave al aparecer/desaparecer
  • Click hace scroll suave al inicio
  
  JavaScript:
  
  window.addEventListener('scroll', function() {
      if (window.pageYOffset > 300) {
          backToTopButton.classList.add('show');
      } else {
          backToTopButton.classList.remove('show');
      }
  });

┌─────────────────────────────────────────────────────────────┐
│ HTML/CSS                                                     │
└─────────────────────────────────────────────────────────────┘

✓ HTML Semántico
  • <nav>, <header>, <footer>, <main>, <section>
  • Atributos aria-* para accesibilidad
  • Breadcrumbs con atributo aria-label

✓ Bootstrap Grid System
  • Clases: container, row, col-*
  • Breakpoints: col-sm-, col-md-, col-lg-
  • Componentes: navbar, card, carousel, offcanvas

✓ CSS Flexbox
  • display: flex
  • justify-content, align-items
  • flex-direction

✓ Position
  • static (default)
  • relative (referencia para absolute)
  • absolute (posición respecto al padre)
  • fixed (posición fija en viewport)

✓ Transiciones y Transformaciones
  • transition: propiedad duración función
  • transform: translateY(), scale(), rotate()
  • Efecto hover con :hover

✓ Animaciones CSS
  • @keyframes nombreAnimacion { ... }
  • animation: nombre duración función delay
  • animation-fill-mode, animation-delay

✓ Selección de Elementos DOM

  • document.querySelectorAll()
  • getElementById()

✓ Event Listeners
  • addEventListener('click', function)
  • addEventListener('scroll', function)
  • addEventListener('mouseenter', function)
  • preventDefault() para evitar comportamiento default

✓ Manipulación del DOM
  • element.classList.add(), remove(), toggle()
  • element.textContent

✓ Temporizadores
  • setTimeout(función, milisegundos)
  • setInterval(función, milisegundos)

✓ Scroll

│ PRÓXIMOS PASOS RECOMENDADOS (EN ORDEN)                      │

PASO 1: PUBLICAR FRONTEND ACTUAL (GRATIS)
  
  Opciones:
  • GitHub Pages
  • Netlify
  • Vercel
  
  Tiempo: 1-2 horas
  Beneficio: Tu sitio estará online y podrás compartirlo


PASO 2: APRENDER BACKEND (Node.js + Express)
  
  ¿Por qué Node.js?
  • Usas JavaScript (mismo lenguaje que frontend)
  • Gran comunidad y recursos
  • Perfecto para APIs REST
  
  Qué aprender:
  1. Node.js básico
  2. NPM (gestor de paquetes)
  3. Express.js (framework web)
  4. Rutas y controladores
  5. Middleware
  
  Recursos:
  • nodejs.org/docs
  • expressjs.com
  • youtube.com/results?search_query=node+express+tutorial
  
  Tiempo estimado: 2-4 semanas


PASO 3: APRENDER BASES DE DATOS
  
  Recomendación: PostgreSQL (SQL)
  
  Qué aprender:
  • Qué es una base de datos relacional
  • Tablas, columnas, tipos de datos
  • Claves primarias (Primary Keys)
  • Relaciones (Foreign Keys)
  • Consultas SQL básicas (SELECT, INSERT, UPDATE, DELETE)
  • JOINs para relacionar tablas
  
  Diseño de tu base de datos:
  
  TABLA: productos
    • id (PRIMARY KEY)
    • nombre
    • descripcion
    • precio
    • stock
    • imagen_url
    • categoria
  
  TABLA: usuarios
    • id (PRIMARY KEY)
    • nombre
    • email (ÚNICO)
    • password_hash (NO guardar en texto plano)
    • rol (cliente/admin)
  
  TABLA: pedidos
    • id (PRIMARY KEY)
    • usuario_id (FOREIGN KEY → usuarios.id)
    • fecha
    • total
    • estado (procesando/enviado/completado)
  
  TABLA: pedidos_items
    • id (PRIMARY KEY)
    • pedido_id (FOREIGN KEY → pedidos.id)
    • producto_id (FOREIGN KEY → productos.id)
    • cantidad
    • precio_unitario
  
  Recursos:
  • postgresql.org/docs
  • postgresqltutorial.com
  • youtube.com/results?search_query=postgresql+tutorial
  
  Tiempo estimado: 2-3 semanas


PASO 4: CONSTRUIR LA API REST
  
  Endpoints a crear:
  
  PRODUCTOS:
    • GET /api/productos → Listar todos
    • GET /api/productos/:id → Ver uno específico
    • POST /api/productos → Crear (solo admin)
    • PUT /api/productos/:id → Editar (solo admin)
    • DELETE /api/productos/:id → Eliminar (solo admin)
  
  AUTENTICACIÓN:
    • POST /api/auth/registro → Crear cuenta
    • POST /api/auth/login → Iniciar sesión (devuelve JWT)
  
  PEDIDOS:
    • POST /api/pedidos → Crear pedido
    • GET /api/pedidos → Ver mis pedidos (usuario autenticado)
    • GET /api/admin/pedidos → Ver todos (solo admin)
  
  Seguridad:
    • Hashear contraseñas con bcrypt
    • Autenticación con JWT (JSON Web Tokens)
    • Validación de datos con express-validator
    • Middleware para verificar roles (admin vs cliente)
  
  Tiempo estimado: 3-4 semanas


PASO 5: CONECTAR FRONTEND CON BACKEND
  
  Modificar tu JavaScript actual:
  
  ANTES (estático):
  // Productos hardcodeados en HTML
  
  DESPUÉS (dinámico):
  // Cargar productos desde API
  fetch('/api/productos')
    .then(res => res.json())
    .then(productos => {
      productos.forEach(producto => {
        // Crear tarjeta HTML dinámicamente
        crearTarjetaProducto(producto);
      });
    });
  
  Qué cambiar:
  • Catálogo: Cargar productos desde API
  • Carrito: Guardar en localStorage + enviar a API al comprar
  • Login/Registro: Enviar datos a API
  • Perfil: Mostrar datos del usuario desde API
  
  Tiempo estimado: 2-3 semanas


PASO 6: CREAR PANEL DE ADMINISTRACIÓN
  
  Nuevas páginas (solo accesibles para admin):
  • /admin/dashboard.html → Resumen de ventas
  • /admin/productos.html → Gestión de productos
  • /admin/pedidos.html → Ver y gestionar pedidos
  
  Funcionalidades:
  • Subir fotos de productos (multer en backend)
  • Crear/editar/eliminar productos
  • Ver lista de pedidos
  • Cambiar estado de pedidos
  
  Tiempo estimado: 2 semanas


PASO 7: SISTEMA DE PAGOS
  
  Opciones (por región):
  • Argentina/Uruguay: Mercado Pago
  • Internacional: Stripe, PayPal
  
  Proceso:
  1. Usuario llega al checkout
  2. Frontend envía pedido a tu backend
  3. Backend crea pedido "pendiente" en DB
  4. Backend inicia transacción con pasarela de pago
  5. Usuario redirigido a página de pago segura
  6. Pasarela confirma pago a tu backend (webhook)
  7. Backend actualiza pedido a "pagado"
  
  IMPORTANTE:
  • Tu sitio NUNCA ve los datos de la tarjeta
  • Todo se procesa en servidores de la pasarela
  
  Tiempo estimado: 1-2 semanas


PASO 8: DEPLOY (PUBLICAR TODO)
  
  FRONTEND + BACKEND + BASE DE DATOS
  
  Opciones recomendadas:
  
  A) Render (más fácil para principiantes)
     • Frontend: Sitio estático (gratis)
     • Backend: Web Service (gratis con límites)
     • Base de datos: PostgreSQL (gratis con límites)
  
  B) Railway
     • Similar a Render
     • Interfaz muy amigable
  
  C) VPS (DigitalOcean, Linode)
     • Más control pero más complejo
     • Costo: $5-10 USD/mes
  
  Dominio:
  • Comprar en Namecheap, GoDaddy
  • Costo: $10-15 USD/año
  • Apuntar DNS a tu servidor
  
  SSL (HTTPS):
  • OBLIGATORIO para producción
  • Usar Let's Encrypt (gratis)
  • La mayoría de plataformas lo incluyen automático
  
  Tiempo estimado: 1 semana
